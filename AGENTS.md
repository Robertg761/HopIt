# HopIt agent rules

## Interface design

- Never use decorative eyebrow text, eyebrow badges, pill-shaped promotional labels, or small pre-heading callouts in HopIt interfaces. This includes patterns such as “One workspace, every device,” “Most useful,” “Available for macOS and Linux,” and uppercase or monospace category labels placed above a heading.
- Put essential context directly in the heading or supporting body copy instead of floating it in a chip or pre-heading label.
- Pill badges are permitted only when they communicate a real, dynamic product state in the authenticated application, such as a role, sync state, security state, quota state, or job status. They must never be used as decoration or marketing emphasis.
